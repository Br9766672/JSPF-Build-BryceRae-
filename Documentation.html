Documentation for Element 5

What is in Element 5

Element 5 is made up of two different scenes a Light world that is made to look like beach and a dark world that is a dark void that only contains death.

Shared Features

In both scenes there is a aninmated player character that can move around scene and interact with physics objects scattered about.

Both scenes also include a sky box to help with the theme of each location. Along with camera constraints to prevent users form looking past the skybox or in the case of the light world. Under the scene map.

Light world

In the light world the plamtrees are sprites that are randomly placed across the sand texured ground to create the illusion of being on a tropical island.

The scene also has a sphere made to look like a beach ball that the player can roll around, and to prevent the ball form falling off the map. There are invisible boxes acting as walls to prevent the sphere from falling.

Dark World

In the dark scene the ground has been made invisible to make it look more like you are in an empty void. The ground is still functional however so the skulls with physics don't all just immediately fall and disappear

File Breakdown

The code has three main files one is the index file. And the two Bablyon scene files.

Index

The index file is responsable for importing and intitalisng all of the other nesscssary files and pulgins for the scenes to work. It is also the file that links all these files together and allows them to communicate with each other, and futhermore allows the scenes to be displayed on a webpage.

The index in element 5 is also responsable for managing the changing between the two scenes.

Scene Files

The scene files are where all of the code for how the scene should look are, and is what actually creates the 3D scenes and how they function

each scene file can be broken down into 3 main sections:

  1. Importing
  2. Functions
  3. Rendering

Importing

The importing section is all about importing all your depedencies for your code to work like classes,libraries,and pulgins including Bablyon itself.

Functions

This is section is where the bulk of the code is and is about creating functions to create your scene.

In this section you create functions on how to:

  • Create or import meshes
  • Applying textures/materials
  • Sprites
  • Animations
  • Player input
  • Lighting and shadows

Rendering

After you have finished with creating the functions, you now move onto the last step of making the scene rendering

In rendering you know use all of the functions to create the scene itself, this is done by tweaking the input values of your functions.

As an example if you created a function that made a box and allowed the position of the box to be assigned, you could then in the rendering phase change the position vaules of the box to precisely place the box anywhere you want in the scene
